Техническое задание

Система создания карточек книг для маркетплейсов

1. Общая цель системы

Разработка приложения для автоматизированного формирования карточек товаров (б/у книг) для маркетплейсов на основе фотографий.

Основные задачи системы:

1. Формирование карточек товара для маркетплейсов (на первом этапе — Ozon).

2. Автоматическое извлечение информации из фотографий книги.

3. Формирование базы книг с фотографиями.

4. Учет статистики работы сотрудников (фотографов / операторов).

5. Частичная автоматизация заполнения карточки товара.

2. Пользователи системы

2.1 Типы пользователей

Оператор (фотограф / сотрудник)

Основные функции:

● создание карточек книг

● загрузка фотографий

● проверка и корректировка данных

Администратор

Функции:

● управление пользователями

● настройка системы

● просмотр статистики, генерация отчетов

● управление базой книг

3. Авторизация и регистрация

3.1 Авторизация

При входе в приложение пользователь проходит авторизацию:

Поля:

● Телефон/email

● Пароль

3.2 Регистрация пользователя

Если пользователя нет в системе:

Создание возможно двумя способами:

1. Через администратора

2. Через приложение с подтверждением администратора

При регистрации указываются:

● ФИО

● телефон

● email

● пароль

4. Создание карточки книги

При создании карточки оператор вводит:

4.1 Основные данные

● Номер коробки

○ используется для формирования артикула товара

● Фотографии книги

○ до 10 фотографий

○ порядок можно менять

○ фото можно заменять

Обязательные фотографии:

1. Фото 01 — обложка книги с линейками для определения размеров

2. Фото 02 — страница с информацией о книге

4.2 Цена

Цена определяется следующим образом:

1. система ищет аналогичный товар на Ozon

2. вычисляется средняя цена

Если цена не найдена: используется значение по умолчанию

5. Распознавание информации с фотографий

Для извлечения информации используется:

● OCR

● нейросети

5.1 Фото 01 (обложка)

Извлекаются:

● размеры книги (по линейкам)

● название книги (если отсутствует на фото 02)

● автор (если отсутствует на фото 02)

5.2 Фото 02 (информация о книге)

Извлекаются:

● ISBN

● название

● автор

● длина

● ширина

● высота

● вес нетто

● вес брутто

● издательство

● год выпуска

● тип бумаги

● тип обложки

● количество страниц

● аннотация

Если название не найдено, выполняется поиск по ISBN.

6. Формирование карточки товара (Ozon)

6.1 Фотографии

● 2 обязательные

● до 5 дополнительных

● возможность изменения порядка

6.2 Поля карточки товара

Основные поля

● Название

Источник:

1. фото 02

2. фото 01

3. поиск по ISBN

● Категория

Фиксированная:

Книги

→ Букинистические издания (1942–2010)

→ Печатная книга

● Артикул товара

Формат:

НомерКоробки_УникальныйКод

Уникальный код генерируется автоматически.

● Цена

1. поиск на Ozon

2. расчет средней цены

Если не найдено: 0

● Габариты

Источник:

● фото 01 (по линейкам)

● фото 02

Если высота не найдена: 35 мм

● Вес с упаковкой

Источник:

● вес брутто

Если не найдено: 450 г

Дополнительные поля

|   |   |
|---|---|
|Поле|Источник|
|Бренд|издательство|
|Автор на обложке|фото 02 → фото 01|
|Направление|проза|
|Хэштеги|вручную или генерация нейросетью|
|Аннотация|фото 02|

В начале аннотации автоматически добавляется текст:

ВНИМАНИЕ! Книга не новая! Состояние - на фото.

При необходимости мы можем добавить больше фотографий для оценки состояния экземпляра.
